# Clawdo Roadmap

This roadmap outlines the incremental development plan for Clawdo.

Clawdo prioritizes **execution safety, auditability, and control**
over rapid feature expansion.

---

## Phase 1 — Architecture & Design (Current)

- System architecture definition
- Execution flow modeling
- Clear separation of decision and execution layers
- Documentation-first development

Status: **In progress**

---

## Phase 2 — Core Interfaces

- State Collector interface definition
- Decision Advisory input/output schema
- Rule Engine constraint model
- Executor action contract

Status: **Planned**

---

## Phase 3 — Reference Implementation

- Minimal executor implementation
- Rule engine reference logic
- Logging and reporting pipeline
- Deterministic replay support

Status: **Planned**

---

## Phase 4 — Integrations & Use Cases

- External advisory model adapters
- Execution environment integrations
- Example automation scenarios
- Security review and hardening

Status: **Future**
